# 山本内科クリニック ホームページ制作プロジェクト

## プロジェクトの方針
- 個人クリニック(内科・訪問診療・内視鏡)のホームページ
- PHPで作り、ヘッダー・フッターなど共通部分は `partials/` フォルダに部品化する(include)
- CSSは共通ルール `assets/css/common.css` ＋ ページごとの `assets/css/ページ名.css` に分ける
- 画像はWebP対応で軽くする(写真は未提供のため、当面は仮のプレースホルダー画像で進める)
- スマホ・PC両対応(レスポンシブ)
- 依頼者はホームページ制作の初心者。専門用語はそのつど簡単に説明する
- 区切りのよいところでGitコミットして記録する

## デザイン方針
- イメージカラー: 白・深青(#2c4a6e系)・グレーの落ち着いたトーン(差し色に落ち着いたグリーンをCTAボタンなどに使用)
  - 参考: 里村クリニック(satomura-clinic.com)の配色・構成の「印象」を参考にしたが、デザイン・コード・文章は独自に作成(丸ごと模倣はしない)
- 印象: 清潔感・信頼感のある内科クリニックらしいトーン
- 内科は高齢の患者さんも多いため、文字はやや大きめ・コントラストはっきりめにする
- ロゴは既存サイトのものを流用(未取得の間はテキストロゴで仮対応)
- トップページのヒーローは、外観+待合室の写真を大きく並べ理念テキストを重ねる構成(3-2参照)
- 診療案内など主要ページへの導線は、左に太いアクセントバー＋矢印付きの「バナー式」カード(.shinryo-card、common.cssで共通定義)
- スマホでは画面下部に電話・ご来院の方への固定ボタン(partials/floating_cta.php、768px以下でのみ表示)
- サイト全体(見出し・本文・ナビ・フッターすべて)をGoogle Fonts「Zen Maru Gothic」で統一(本文400、見出し600、ヒーローh1のみ700)

    同じ丸ゴシックの雰囲気で無料・商用利用可能なZen Maru Gothicを採用(partials/head.phpでGoogle Fontsを読み込み、common.cssのbody/h1,h2,h3に指定)

- content-section内のh2は、左のアクセントバーではなく中央下の短い下線アクセント(::after)に変更。cta-box内のh2にも白系の下線アクセントを追加

## 症状・病名の個別解説ページ(68ページ、対応済み)
- shojo.php・byomei.phpの全タグ(症状39件＋病名29件)に、それぞれ専用の解説ページを作成
  - データは data/glossary.php に一元管理(key, type, category, label, related, lead, points, advice)。
    ページ本体は partials/glossary_render.php が共通描画し、実ファイル(/shojo/{key}.php, /byomei/{key}.php)は
    `$item_key` をセットして glossary_render.php を include するだけの薄いラッパー(68ファイルはPHP CLIスクリプトで自動生成)
  - 各ページの構成: パンくず → 見出し(症状・病名名) → リード文 → 「考えられる原因(症状側)/主な原因・特徴(病名側)」の箇条書き →
    「受診の目安/当院での対応」→ 診断ではない旨の注記 → 関連する診療案内へのリンク → 予約バナー
  - 内容は一般的な医学知識の範囲にとどめ、診断確定を避ける言い回し(「〜のことがあります」等)。特に胸の痛みなど緊急性のある症状には
    救急受診も検討するよう注記
  - shojo.php/byomei.phpのタグは、直接shinryo/*.phpへ飛ばすのではなく、まずこの解説ページへリンクする形に変更
- partials/booking_banner.php: 全68ページの下部に表示する予約バナー。電話予約(tel:リンク、機能する)とWeb予約(まだシステムがないため
  「準備中」の非活性ボタン表示)の2つを設置
- sitemap.xmlに68ページ分のURLを追加(合計89件)

## 医療広告ガイドライン上の注意
- 「必ず治る」「地域No.1」等の誇大・断定・比較優良表現、体験談は使わない
- 資格・専門性の記載は伺った内容(院長・副院長の資格名)の範囲でのみ正確に記載する
- 「総合病院と連携」「訪問診療で看取りまで対応」は事実の記載としてOK
- 迷ったら「客観的に証明できるか」「過度な期待を持たせないか」で判断し、載せない方向を優先する
- 胃カメラ・大腸カメラページは、おなかとおしりのすずきクリニック(onakatooshiri.com)の構成を参考にしたが、
  文章はすべて独自作成。かつ同院固有の設備・実績(NBI搭載内視鏡、42,000件以上の実績、麻酔科標榜医、
  炭酸ガス送気、早朝検査、Web予約、費用の具体額 等)は山本内科クリニックで未確認のため一切記載していない。
  実際に確認できた「鎮静剤・麻酔剤使用」「大腸ポリープの日帰り切除」「土曜日の内視鏡検査」のみ記載。
  検査費用はページに掲載していない(掲載する場合は具体的な金額を確認してから追記する)

## 画像のWebP対応・サイズ自動付与(CLS対策)ヘルパーを作成
- 「画像をWebP対応で表示したい。使える環境ではWebPを、使えなければ元のjpg/pngを表示する仕組みを
  共通の関数として作り、各ページはその関数を通す形に統一したい。width/heightも自動で付けてCLSを
  防ぎたい」との要望を受け、partials/image_helper.php を新設した
  - 一言でいうと: `site_image('/assets/images/foo.jpg', '説明文', 'css-class')` と書くだけで、
    (1) ブラウザがWebPに対応していれば軽いWebP画像を、対応していなければ元のjpg/pngを自動で
    出し分け(`<picture><source type="image/webp">`+`<img>`のブラウザネイティブな仕組みを利用。
    JS判定やサーバー側のUA判定は不要)、(2) 画像の実際の縦横サイズを自動で読み取ってwidth/height
    属性を付ける(読み込み中に他の要素がガタッとずれるCLSを防止)、という2つを済ませてくれる
  - WebP画像は初回アクセス時にPHPのGD拡張(imagewebp、量質82)でその場から自動生成し、元画像と
    同じ場所に `.webp` として保存する。2回目以降はキャッシュされたWebPファイルをそのまま使う
    (元画像より新しい場合のみ、filemtimeを比較して再生成する)。ビルドツールやnpm等は一切不要
  - 幅・高さはgetimagesize()で画像ファイルから直接読み取るため、ページ側でサイズを手入力する
    必要がなく、実寸を画像ファイル自体に一元管理できる(1リクエスト内では同じ画像を何度呼んでも
    再読み込みしないようstaticキャッシュも実装)
  - GD拡張が無い環境や、書き込み権限が無い環境でも、WebP生成に失敗した場合は静かに諦めて元画像の
    `<img>`のみを出力するようフォールバックする設計(エラーや警告でページが壊れることはない)
  - partials/head.phpの先頭でrequire_onceし、全ページで自動的に使えるようにした
  - サイト内の全22箇所の`<img>`タグ(index.php 9件、shinryo/naishikyo.php 4件、
    shinryo/ikamera.php・daichokamera.php 各2件、shinryo/index.php・clinic.php・egao.php・
    tanpopo.php・partials/header.php 各1件)をsite_image()呼び出しに置き換えた。
    ヒーロー写真とヘッダーロゴ(常に画面上部に表示される画像)は`loading="eager"`を明示指定し、
    それ以外の画像はデフォルトの`loading="lazy"`のままにした(表示速度とCLS対策の両立)
  - 元画像は素材として最大2.4MBのPNGなどサイズが大きかったが(合計約23MB)、WebP変換により
    合計約1MBまで縮小(約92%削減)を確認。ブラウザのNetwork Requestsで実際にWebPが読み込まれる
    ことも確認済み。GD拡張・WebPサポートはローカル環境で確認済み(gd_info()のWebP Support: 1)
  - 生成される`.webp`ファイルはGit管理せず、.gitignoreに`assets/images/*.webp`を追加した
    (元画像さえあればsite_image()がその場で再生成できる派生ファイルのため)
  - 【注意点・実装中に踏んだ落とし穴】 コメント内に説明用として`<?php ... ?>`という例示コードを
    書いたところ、PHPの`//`コメントは行内に`?>`が現れるとそこでPHPモードごと終了してしまうという
    仕様があり、それ以降のコード全体(関数定義含む)が実行されずそのままページに出力される事故が
    発生した。コメント内では実際のPHPタグを書かず、関数呼び出しの例だけをプレーンテキストで
    示すよう修正して解消した
